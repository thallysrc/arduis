```python
# review.py (GTK-free, no gi)
def argv_diff(repo: str) -> list[str]:
    return ["git", "-C", repo, "--no-pager", "diff"]
def argv_diff_stat(repo: str) -> list[str]:
    return ["git", "-C", repo, "--no-pager", "diff", "--stat"]
def argv_status_porcelain(repo: str) -> list[str]:
    return ["git", "-C", repo, "status", "--porcelain"]
def parse_porcelain_clean(stdout: str) -> bool:
    return stdout.strip() == ""           # empty => clean => safe to remove  [VERIFIED on host]
def argv_worktree_remove(source_repo: str, worktree_dir: str) -> list[str]:
    return ["git", "-C", source_repo, "worktree", "remove", worktree_dir]   # NO --force
def argv_worktree_prune(source_repo: str) -> list[str]:
    return ["git", "-C", source_repo, "worktree", "prune"]
def argv_current_branch(repo: str) -> list[str]:
    return ["git", "-C", repo, "rev-parse", "--abbrev-ref", "HEAD"]
def argv_ahead_behind(repo: str, branch: str) -> list[str]:
    return ["git", "-C", repo, "rev-list", "--left-right", "--count", f"{branch}...{branch}@{{u}}"]
def parse_ahead_behind(stdout: str) -> tuple[int, int]:
    parts = stdout.split()
    if len(parts) == 2 and parts[0].isdigit() and parts[1].isdigit():
        return int(parts[0]), int(parts[1])
    return (0, 0)
```

<plan_decisions>
- **review.py is pure argv builders + parsers, GTK-free** — imports nothing beyond stdlib (it
  needs none here; do NOT import `gi`, `os` is unnecessary). The test asserts `"import gi"` and
  `"from gi"` are not in the source. Mirrors `worktree.py` exactly.
- **The never-force rule is structural, not a runtime check.** `argv_worktree_remove` simply does
  not build a `--force`/`-f` token. The test asserts `"--force" not in argv and "-f" not in argv`
  for several `(source_repo, worktree_dir)` inputs including ones with a leading dash in the dir
  name (the discrete-element argv defends against flag injection — but DO NOT add `--` separator
  unless the executor confirms `git worktree remove` accepts it; the research argv is verified
  without it). Keep the argv exactly as in `<interfaces>`.
- **`parse_porcelain_clean(stdout)` returns `stdout.strip() == ""`** — empty porcelain ⇒ clean ⇒
  safe to remove. Any line (`?? a.txt`, ` M file`, `A  staged`) ⇒ dirty ⇒ False. This is the gate
  the conclude orchestrator (Wave 3) consults per repo before any removal.
- **`parse_ahead_behind`** tolerates: `"3\t2"` → `(3,2)`; `"0\t0"` → `(0,0)`; `""` → `(0,0)`;
  `"garbage"` → `(0,0)`; a single token → `(0,0)`. Never raises (a repo with no upstream makes
  `git rev-list ...@{u}` fail rc!=0 with empty/garbage stdout — the caller treats `(0,0)` as
  "no tracking info" and the window suppresses the arrows; pin that the parser itself never throws).
- **`review_cache.py` shape (PIN this — Wave 2 calls it):** a tiny class
  `ReviewCache` holding `dict[str, tuple[object, float]]` keyed by `task_id`:
  ```python
  class ReviewCache:
      def __init__(self) -> None:
          self._d: dict[str, tuple[object, float]] = {}
      def put(self, task_id: str, payload: object, now: float) -> None:
          self._d[task_id] = (payload, now)
      def get(self, task_id: str) -> tuple[object, float] | None:
          return self._d.get(task_id)
      def fresh_payload(self, task_id: str, now: float, ttl: float) -> object | None:
          rec = self._d.get(task_id)
          if rec is None or not is_fresh(rec[1], now, ttl):
              return None
          return rec[0]
  ```
  Plus the module-level `is_fresh(ts, now, ttl)`. `now` is supplied by the caller (the window passes
  `time.monotonic()`); the cache is monotonic-agnostic so tests inject fixed floats. **Two TTL
  constants live here for Wave 2 to import:** `GIT_TTL_S = 30.0`, `GH_TTL_S = 120.0` (08-RESEARCH §4
  defaults; tunable after dogfooding).
- **GTK-free (the discipline):** both files import only stdlib; the tests assert no `gi` import.
- **TDD:** write the failing tests first (RED), implement the pure functions (GREEN). No refactor
  cycle needed — these are flat pure functions.
</plan_decisions>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| branch name / worktree dir → git argv | the branch is user-chosen and the worktree dir is derived from it; both cross into a git invocation as discrete argv elements. |
| `git diff`/`git status` output → display only | git output is shown read-only (a VTE leaf / a subline); it is NEVER eval'd or used to build a path/command. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Tampering / EoP | branch/worktree-dir injected into a git argv | mitigate | every argv is a Python LIST with the repo/dir/branch a DISCRETE element (mirrors worktree.py T-02-01); no shell, no `shell=True`, nothing joined into a string. The Wave-2 caller routes argv through HostRunner. |
| T-08-02 | Tampering (data loss — the cardinal sin) | `argv_worktree_remove` could force-delete uncommitted work | mitigate | `argv_worktree_remove` STRUCTURALLY never emits `--force`/`-f`; the never-force guard test pins it. The dirty-tree gate (`parse_porcelain_clean`) is the precondition the Wave-3 orchestrator checks before any removal (criterion 4 / D-04). |
| T-08-03 | DoS (self) | unbounded git/gh reads exhaust resources / gh rate limit | mitigate | the TTL cache (`is_fresh` + ReviewCache) is the throttle backbone (GIT-01); Wave 2 gates reads on it + an in-flight debounce. This plan provides the mechanism; no poll exists. |
</threat_model>
