<plan_decisions>
- **Idle threshold (dot):** `idle_minutes = 10` config default (D-03 says IDLE = ready +
  threshold but pins no number; claude's own idle_prompt fires at ~60s, so 10 min is a
  visibly-distinct "you forgot about me" tier). Configurable under `[attention]`.
- **Stale-running ceiling (Pitfall 7 guard from CONTEXT specifics):** `running` older than
  `RUNNING_STALE_CEILING_S = 2 * 3600` (a "generous ceiling") degrades to ready on the sweep
  — generous because a long tool call emits no events (Pitfall 6) and a wrongly-degraded
  running could chain into auto-suspend. Dead hook-pid/pgid → `ended` is the hard signal.
- **Config keys (D-11 + research OQ 2/5):** `[attention] auto_suspend_minutes = 0` (0/absent
  = OFF), `idle_minutes = 10`, `notify_ready = false`, `sound = false`.
- **Declined-consent persistence (D-02):** a marker file
  `~/.local/share/arduis/hooks_declined` (touch on decline; presence suppresses the dialog).
  tomllib is read-only so the marker avoids growing a TOML writer dep.
- **Aggregation of ENDED (D-06):** precedence waiting > running > ready > idle > ended;
  terminals whose status is None (no state file ever — plain shells, pre-SessionStart agents)
  are excluded entirely (Pitfall 8). Empty input → None ("no opinion").
- **Wave-1 ordering vs 04-01:** this plan shares wave 1 with 04-01 (no file overlap). If the
  orchestrator executes wave 1 SEQUENTIALLY it MUST run 04-01 before 04-02 so the
  `hook_script_source()` content test runs unskipped; under parallel execution the
  `pytest.mark.skipif` guard in Task 2 covers the window and the test goes green at the
  wave merge (full-suite gate).
</plan_decisions>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| term_id (contains user branch name) → filesystem filename | branch slashes/dots become a state-file leaf |
| on-disk state file (hook-written) → arduis parse | half-written/hostile JSON crosses into the reader |
| user's ~/.claude/settings.json → merge builder → written back | the user's load-bearing config flows through arduis code |
| arduis.toml (user-edited) → config values | typed config drives a process-killing feature (auto-suspend) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-06 | Tampering/EoP | state_file_path traversal via branch-derived term_id | mitigate | Character allowlist + ".." collapse; tested: hostile id yields a flat leaf whose dirname == status dir |
| T-04-07 | DoS | partial/garbage state file crashing the reader on the main loop | mitigate | read_state returns None on ANY parse failure (tested: empty, garbage, missing-key) |
| T-04-08 | Tampering | merge corrupting/normalizing the user's settings (data loss in a dense, load-bearing file) | mitigate | Pure deepcopy append-only merge; idempotent; dedupe by script path; existing entries asserted byte-identical in tests; the WRITE (backup + atomic) is Plan 03's, building on this tested fn |
| T-04-09 | DoS (process kill) | should_autosuspend firing on a working agent | mitigate | Policy only ever True for READY/IDLE/ENDED aggregates; RUNNING/WAITING/None tested immune at any age; minutes=0 default-off |
| T-04-10 | Tampering | hostile arduis.toml values (huge/negative/strings) | mitigate | Typed parse with per-key fallback to defaults; negative minutes treated as 0 (off) |
| T-04-11 | Info disclosure | clear_status_dir following a symlink out of the status dir | mitigate | unlink only regular non-symlink direct children (tested: symlinks and subdirs ignored) |
</threat_model>
