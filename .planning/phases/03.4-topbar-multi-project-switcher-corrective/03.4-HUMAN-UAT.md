cause: (1) only `__init__` seeded the launch project's main scratch shell — `_switch_project` never spawned a freshly-opened project's shell; (2) `_open_project`/`_init_projects` skipped the degenerate single-repo member fallback (members=[]).
fix: `4a93988` — `_switch_project` seeds the main shell on first activation; new `_resolve_members()` adds the degenerate fallback. Regression test added; suite 427 passed, switch smoke 5/5.
