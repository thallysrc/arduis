the topbar as switchable PROJECT tabs + a "+ Abrir projeto" folder picker, wire `projects.json`
persistence + launch auto-register (D-07), and make project switching "both alive" (D-08) by retaining
each background project's GTK leaf/terminal/layout maps and only re-reflecting the active project's tree.

has its own grep-verifiable acceptance gate and `main` stays launchable between them. It does NOT touch
teardown/remove/cap-union wiring (that is Plan 04) — switching never tears anything down here.

Purpose: deliver SC-1 (tabs swap sidebar+workspace), SC-2 (open + persist across restart), SC-3 (both
alive on switch), SC-4 (launch auto-registers cwd), SC-6 (real Project model encapsulates state), and

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user → Gtk.FileDialog | user picks an arbitrary host folder that arduis will scan + later spawn shells/compose in |
| project root/basename → GTK labels + git/scan argv | filesystem-derived strings reach UI labels and command argv |
| branch-derived term_id → status-file path | per-project term_id namespacing prevents cross-project status bleed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03.4-07 | Tampering | project tab labels | mitigate | `Gtk.Label.set_text(project.name)` only — never `set_markup` on a filesystem-derived basename (T-03-09 / V5 label-injection guard) |
| T-03.4-08 | Tampering | detect_member_repos / _scan_tasks on a chosen root | mitigate | scanned dir names only ever reach git argv as discrete `-C <path>` list elements via HostRunner.wrap_argv (existing T-03.2-04 discipline); no shell strings |
| T-03.4-09 | Spoofing/Elevation | "+ Abrir projeto" arbitrary dir | accept | same trust model as today's launch dir — the user chose it; arduis only reads git/scan + spawns the user's own shell there (RESEARCH security domain (b)) |
| T-03.4-10 | (resource leak) | switch retains background terminals (D-08) | accept | intentional RAM tradeoff bounded by the global cap (D-09, wired Plan 04) + per-task hibernate; PO-decided |
| T-03.4-17 | Spoofing-of-state | cross-project status-file collision | mitigate | `project_term_id()` wraps the term_id at all 4 status-file call sites + the watcher key (Task 1c) so two projects' same-named branches never share a status file (A3) |
</threat_model>
