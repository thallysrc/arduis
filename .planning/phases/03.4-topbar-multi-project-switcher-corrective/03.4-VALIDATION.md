## Sampling Rate

- **After every task commit:** Run the quick command for the touched module.
- **After every plan wave:** Run the full suite (`python -m pytest -q`).
- **Before `/gsd-verify-work`:** Full suite green + headless broadway smoke for switch/teardown.
- **Max feedback latency:** ~30 seconds (unit).

## Manual-Only Verifications

| Behavior | Decision/SC | Why Manual | Test Instructions |
|----------|-------------|------------|-------------------|
| Topbar lists projects; clicking a tab swaps sidebar+workspace | SC-1 | Real GTK render/input | Open 2 projects, click tabs, confirm sidebar(tasks)+workspace swap |
| "Open project" picker registers + persists a root across restart | SC-2 | FileDialog + relaunch | Open a multi-repo root, restart app, confirm it's still listed + selected |
| "Both alive": switching projects leaves other project's agents/containers running | SC-3/D-08 | Process-tree inspection | Start agents in proj A, switch to B, `ps`/`pgrep` confirm A's pgids + containers still up |
| Removing a project tears down its live tasks (confirm dialog) | D-10 | Process-tree + compose | Remove a project with live tasks; confirm SIGHUP→SIGKILL pgids + `docker compose down -v`; no orphans |
| App-exit tears down ALL projects (no orphans) | D-11 | Process-tree | Open 2 projects with live tasks, close window, `pgrep`/`docker ps` confirm zero orphans |
