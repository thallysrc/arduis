<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filesystem → projects.json | user-chosen project roots (paths) are written/read as app state |
| projects.json → term_id/status-file paths | branch-derived ids feed `attention.state_file_path` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03.4-01 | Tampering | projects_store.save_projects | mitigate | atomic mkstemp-in-dir + `os.replace`; mid-write failure unlinks temp, leaves original intact (test_atomic_write_no_partial_on_failure) |
| T-03.4-02 | Tampering/DoS-of-state | projects_store.load_projects | mitigate | tolerant load: bad json / not-a-dict / missing file → `([], None)`; one bad entry never aborts the whole load (test_load_bad_json_returns_empty) |
| T-03.4-03 | Tampering | project_term_id → state_file_path | mitigate | discriminator is sha1-hex (allowlist-safe) prefixed onto term_id; result still passes `sanitize_term_id` (no path traversal); cross-project collision eliminated (A3) |
| T-03.4-04 | Information disclosure | projects.json contents | accept | file holds only directory paths the user already has on disk; no secrets (V6 n/a per RESEARCH) |
</threat_model>
