<objective>
Wire the Wave-1/2/3 modules into `window.py` so a task can opt in to its own isolated docker-compose
stack. Auto-detect a root compose (CONT-01); add a per-task default-OFF "Isolar containers" toggle to
the sidebar row context menu (D-11 — the opt-in IS the consent gate); flipping ON runs the full async
chain (read base ports -> offset+probe -> write override with `ports: !override` -> `up -d`),
persists `ContainerState`, and renders the resolved ports as badges with a toast+spinner during the
op; flipping OFF runs `down --remove-orphans --volumes` keeping the port map; teardown wires into the
hibernate path and app-exit as a SEPARATE channel from the killpg agent teardown (Pitfall 7); and a
startup pass loads each task's state for badges and reconciles orphaned `arduis-*` projects. This is
the single hot file — pure glue over tested modules.

Purpose: This is the only `gi`-touching, behavior-bearing file in Phase 7. It introduces ZERO new
mechanisms: a tolerant read (Plan 02), an async runner (Plan 03), a context-menu entry (the Phase-3
pattern), a toast (new Adw.ToastOverlay), and badges (the existing badge machinery). The crux is the
async ordering (config -> assign -> write -> up -> persist) with partial-failure cleanup, and that
container teardown is its OWN channel (containers are daemon-owned, NOT in arduis's process group).
Output: edits to `src/arduis/window.py`; full suite green (no regression); ready for the Wave-4
headless smoke + live UAT.
