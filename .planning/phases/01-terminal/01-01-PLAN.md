<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| arduis process → host PTY child (`zsh`) | The app spawns a host shell with full user privileges via a direct native PTY (no sandbox in v1). Everything the embedded shell can do, the user can already do — but argv/env construction is the surface where injection or env-poisoning could occur. |
| `_FLATPAK` config flag → execution path | A stubbed v2 branch must never silently execute a half-built `flatpak-spawn` command path in v1. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering / Elevation | `spawn.build_spawn_command` argv assembly | mitigate | Build argv as a Python **list** (`["zsh","-l","-i"]`), never a shell string; no `shell=True`; tested by test_spawn_argv asserting list shape + absence of injection-prone joins. |
| T-01-02 | Tampering | env list (`TERM=xterm-256color`) | mitigate | env is a fixed `KEY=VALUE` list literal merged onto the inherited env; no user-controlled interpolation in Phase 1. Tested for exact value. |
| T-01-03 | Elevation (latent) | `HostRunner` Flatpak stub `_FLATPAK` | mitigate | Flatpak branch raises `NotImplementedError` — cannot run a half-formed escape path in v1. Tested by test_flatpak_stub. |
| T-01-04 | Information disclosure | exit-status decode | accept | `decode_exit` only maps an int→int; no PII, no side effects. Low risk for a local single-user dev tool. |
</threat_model>
