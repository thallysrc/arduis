# Phase 03.3: Topbar Repo Chips (design-alignment, INSERTED) - Context

**Gathered:** 2026-06-14
**Status:** Ready for planning
**Origin:** PO correction during manual testing — GSD's D-06 ("topbar = project name only") diverged
from the approved design (`docs/mockup/interface-v2-bridgespace.png` + `docs/MOTIVATION.md` 3-level
model), which surfaces repo selection in the topbar as visible chips/dots. Full rationale in
`.planning/phases/03.2-projects-and-cross-repo-tasks/03.2-GAP-topbar-repo-chips.md`.
**Mode:** PO-directed (the design choice is the user's; remaining sub-choices take sensible defaults).

<domain>
## Phase Boundary

Surface the project's member repos as **toggleable chips in the topbar** (the mockup affordance),
replacing the bare project-name string from D-06. The toggled-ON set becomes the default repo
selection for new tasks; the active task's repos are reflected in the chips. Member-repo detection
is fixed to exclude linked worktrees so the chip bar is usable on a real root.

**Supersedes:** Phase 03.2 D-06 (topbar = name only, no switcher).
**Out of scope:** a PROJECT switcher (still one arduis = one launch-dir project — a separate future
decision); container badges (those come with Phase 7 / 07-04, designed to sit alongside these chips);
the Command/Swarm/Review room tabs from the mockup (later/out of v1).
</domain>

<decisions>
## Implementation Decisions

- **D-01:** The topbar renders ONE chip per project member repo, each with a status bolinha
  (reuse the existing sidebar dot classes), to the left of / replacing the bare name. The project
  name stays visible (label or a leading element).
- **D-02:** Chips are **toggleable** (click toggles on/off). The toggled-ON set is the DEFAULT
  repo selection seeded into the "Nova task" dialog (the dialog's per-repo checkboxes still exist
  and inherit this set — per-task override preserved). "Select multi-repo in the topbar" without
  losing per-task flexibility.
- **D-03:** When a task is selected/active, its repos are highlighted/reflected in the chips
  (visual state ties the topbar to the visible workspace).
- **D-04 (mandatory sub-fix):** `detect_member_repos` counts only direct subdirs whose `.git` is a
  **directory** (a true repo), EXCLUDING `.git`-FILE linked worktrees (the user's `Livon-Saude`
  root has ~20 `backend-*`/`frontend-*` worktrees that would flood the bar). Preserve the
  degenerate 1-repo case and the existing 03.2 tests (update the ones that asserted `.git`-file
  inclusion, recording the change as intended).
- **D-05:** Chip overflow (a clean meta-repo with many repos): show chips that fit + an overflow
  menu ("+N"). Horizontal scroll rejected (fiddly with VTE focus).
- **D-06:** GTK-free where possible: a small `topbar`/chip-state helper holds the toggled set;
  window.py renders + wires. No new dependency.
</decisions>
