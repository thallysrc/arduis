## Accepted Risks

- **Repo names are local-filesystem-derived plain text.** Member-repo chip
  labels come from `os.scandir` of the project root (the developer's own
  machine), not from any remote/untrusted source. They are rendered exclusively
  via `Gtk.Label.set_text` (no markup parsing) and only ever land downstream as
  discrete `-C <path>` git-argv list elements (03.2 mitigation T-03.2-04, never
  a shell string). A hostile dir name (newline/space/markup metachar) cannot be
  interpolated into markup or a command. Accepted as low-risk given the local
  trust boundary.
- **T-03.3-03 ŌĆö symlink non-resolution.** The `isdir` membership predicate does
  not resolve symlinked subdirs (`follow_symlinks=False`). This is the declared
  accepted disposition; `test_symlinked_subdir_not_followed` guards against
  regression.
