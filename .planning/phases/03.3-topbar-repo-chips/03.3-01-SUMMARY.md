## Public Contract for Wave 2 (window.py, Plan 02)

### `project.detect_member_repos(root) -> list[str]`
Returns sorted names of direct subdirs of `root` whose `.git` is a **DIRECTORY**. A subdir whose `.git` is a FILE (linked worktree / submodule) is NOT a member. Root's own `.git` is never a member. Symlinked subdirs not followed. Returns `[]` on `OSError` or when there are no members (the degenerate 1-repo project the caller special-cases). **Behavior change vs 03.2:** previously counted `.git`-FILE subdirs too — that is now intentionally excluded.

### `topbar.ChipState(members: list[str])`
Pure GTK-free state. window.py constructs ONE instance from `detect_member_repos(...)` output (or the degenerate 1-element list), renders one chip per `.members` entry, and seeds the New-task dialog from `default_selection()`.

| Member / method | Meaning |
|---|---|
| `.members: list[str]` | given member-repo names, order preserved (never mutated) |
| `.selected: set[str]` | toggled-ON DEFAULT subset; starts as ALL members (D-02) |
| `.active_repos: set[str] \| None` | the visible/active task's repos (D-03); `None` = nothing reflected |
| `.toggle(repo)` | flip one member on/off; no-op for non-members; never mutates `.members` |
| `.default_selection() -> list[str]` | toggled-ON members in `.members` order — **seeds the New-task dialog checkboxes (D-02)** |
| `.reflect_active(set \| None)` | set the active-task highlight **without** touching `.selected` (D-03); `None` clears it (pinned `main` / no task → plain default highlight) |
| `.is_selected(repo) -> bool` | toggled-ON visual state for a chip |
| `.is_active(repo) -> bool` | active-task highlight for a chip |

**Default-vs-reflected distinction (D-02 vs D-03):** `.selected` is the durable new-task default (changed only by `toggle`); `.active_repos` is a transient reflection of the visible task (changed only by `reflect_active`). They never alias — reflecting an active task is fully reversible (`reflect_active(None)`).
