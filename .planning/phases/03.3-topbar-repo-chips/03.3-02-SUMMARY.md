# Phase 03.3 Plan 02: Topbar Repo Chips (window.py render + wire) Summary

**Topbar now renders one toggleable Gtk.ToggleButton chip per member repo (dot + name) backed by ChipState, seeds the New-task dialog from the toggled-ON set, reflects the active task's repos, and folds >6 repos into a "+N" overflow menu — packed at one header end so Phase 7 / 07-04 badges fit beside it.**

## Accomplishments
- **D-01 chip render:** `_build_chip_bar` / `_make_chip` build one flat `Gtk.ToggleButton` per member repo, child = a `Gtk.Box` of a `●` dot `Gtk.Label` (reusing `arduis-dot-active`) + a name `Gtk.Label` rendered via `set_text` only (never `set_markup`). The project name stays visible as the centered `Adw.WindowTitle`.
- **D-02 toggle + dialog seed:** clicking a chip mirrors the button's active state into `ChipState.selected`; the New-task dialog's per-repo `Gtk.CheckButton`s are now `set_active(repo in default_selection())` instead of unconditional all-on, with the checks still user-overridable per task (all-on fallback when chip state is missing).
- **D-03 active reflection:** `_reflect_active_chips(sid)` reflects the visible workspace's repos (`reflect_active({repo_names})`, or `None` for pinned main / repo-less task) and restyles; wired into `_swap_workspace`, `_build_task_workspace` (create/resume), and `_show_hibernated_placeholder` — every path that sets `_active_workspace_sid`.
- **D-05 overflow:** module constant `_MAX_VISIBLE_CHIPS = 6`; >6 members render the first 6 inline + a `+N` `Gtk.MenuButton` whose `Gio.Menu` items target the new `win.toggle_chip(repo)` action. No horizontal scroll.
- **07-04 co-existence:** the chip bar is a single `Gtk.Box` packed at `pack_start` after `+New`, leaving `pack_end` + the title region free for Phase 7's container toggle + port badges.
- Full suite green (336 passed); window.py parses + imports clean headless under broadway.

## Helper / API reference (for 03.3-03 smoke + Phase 7 / 07-04)
- `_build_chip_bar()` — clears + rebuilds the bar from `_member_repos`, constructs `self._chip_state = ChipState(members)`, renders ≤`_MAX_VISIBLE_CHIPS` chips + the `+N` overflow menu, calls `_restyle_chips()`. Called once at the end of `_resolve_project`.
- `_make_chip(repo) -> Gtk.ToggleButton` — registers `_chip_btn_by_repo[repo]` + `_chip_dot_by_repo[repo]`.
- `_restyle_chips()` — in-place: sync each button's `active` to `is_selected`, add/remove `arduis-chip-active` by `is_active`. No rebuild.
- `_reflect_active_chips(sid | None)` — the single chip-reflection chokepoint; `None` / `_MAIN_SID` clears, a task reflects `{r.repo_name for r in task.repos}`.
- `win.toggle_chip(repo)` action → `_on_toggle_chip` (overflow-menu path; uses `ChipState.toggle`, a no-op for non-members).
- The chip bar lives at `pack_start` after `_new_btn`; **07-04 should pack its container toggle + port badges at `pack_end`** (free by design).
