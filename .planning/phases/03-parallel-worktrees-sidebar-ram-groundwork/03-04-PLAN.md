<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| git branch name → sidebar/pane label | A branch name (from git output / user dialog input) is rendered as a `Gtk.Label` in the sidebar row and pane header. |
| sidebar row → focus-or-swap action | A user activating a row drives a layout mutation against in-process session_ids. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-09 | Tampering (label injection) | branch name in `Gtk.Label` | mitigate | Use `label.set_text(branch)` (NOT `set_markup`) so a branch name like `<b>x` or with `&`/`%` is rendered literally — no Pango-markup or format-string injection into the sidebar/pane header. |
| T-03-10 | Spoofing | focus-or-swap on an unknown session_id | mitigate | Row activation resolves a session_id from the trusted `row↔sid` map built from `self._store`; `resolve_selection` only ever returns a known visible id or the current focused id — it cannot target a session not in the store. |
| T-03-11 | Denial of Service | unbounded GtkPaned nesting | accept | Splits are user-driven and bounded by the 5–12 working set + the active-agent cap (enforced in Plan 03-05); min pane size 240×120 prevents zero-size panes. No external trigger can force unbounded nesting. |

Repudiation/Info-Disclosure/EoP are N/A here — no auth surface, no secret handling, no privilege change (spawn/teardown are unchanged Phase-2 seams already threat-modeled in 02).
</threat_model>
