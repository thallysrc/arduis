- **Capturing the fed bytes (the load-bearing assertion):** wrap the shell terminal's `feed_child`
  to record what was fed. Two acceptable approaches — pick the one that works against the real
  widget:
    (a) monkeypatch the specific shell `Vte.Terminal` instance's `feed_child` with a recorder that
        appends bytes to a list then (optionally) calls the original; OR
    (b) monkeypatch `arduis.repoconfig.setup_feed_bytes` is WRONG (it's the unit under test) — do
        NOT do this; instead record at the terminal boundary (a).
  Assert the recorded bytes for an accepted repo START with `b"cd "` and contain each fixture
  command line, and that NOTHING was fed to the AGENT terminal (`{task}:t0`) — wrap t0's feed_child
  too and assert it only ever received the agent `claude` bytes (never a `cd ...` setup payload).
- **Driving the dialog headlessly:** `Adw.AlertDialog` cannot be clicked headless. Instead, refactor
  is NOT needed — invoke the response handler directly. After `_run_repo_setups` presents the dialog,
  the smoke cannot click it; so structure the assertions around the SEAMS:
    * absent-file: write NO `.arduis.toml`; call `_run_repo_setups(task)`; assert the shell recorder
      stays EMPTY and (best-effort) no dialog was presented — verify by asserting nothing was fed
      and the trust list is still empty (criterion 1).
    * trust-then-feed: rather than clicking the real dialog, the smoke calls the trust path the
      dialog's "trust" branch runs — i.e. emulate accept by calling `trust.record_trust(...)` for
      the fixture repo's (realpath, hash) THEN `_run_repo_setups(task)` again: now the repo is
      already-trusted so it feeds SILENTLY (no dialog) and the shell recorder receives the
      cd-guarded bytes. This proves both the silent-trusted path AND the feed shape WITHOUT needing
      to click a GTK dialog. (The "dialog shows the exact commands + accept persists" is covered by
      the live UAT — a human reads the dialog text; the smoke proves the persisted-trust→silent-feed
      mechanism and the byte shape.)
    * changed-hash re-prompt: after trusting hash H1, rewrite the fixture `.arduis.toml` with a
      DIFFERENT command set (hash H2), call `_run_repo_setups(task)`; assert the shell recorder did
      NOT receive the new commands silently (is_trusted False for H2) — i.e. a gate would fire. Prove
      it via `trust.is_trusted(path, repo_id, H2) is False` AND the recorder did not get H2's bytes
      without an explicit trust.
    * resume-no-setup: call `_resume_task(task)` (or assert by code path) and confirm the shell
      recorder receives NO setup bytes from resume — resume must never call `_run_repo_setups`.
      If `_resume_task` is awkward to drive fully headless, assert structurally: grep that
      `_run_repo_setups` is called only inside `_finalize_task_creation` (a source assertion in the
      smoke: read window.py, confirm `_resume_task`'s body does not contain `_run_repo_setups`).
  Keep each as a `check(...)`. The smoke MUST exit non-zero if any check fails.
