<objective>
Extend the GTK-free session model so each worktree owns a LIST of terminals
(default 2 — one `agent`, one `shell`) instead of a single pid/pgid. This is the
domain half of the Phase-03.1 pivot (the canvas shows ONE worktree's terminals).
The store stays GTK-free and JSON-serializable; hibernate clears every terminal's
pgid; RAM is summed across terminals by the caller.

This is a TDD plan: tests describe the contract first, then the model satisfies them.

Purpose: Lock the N-terminals-per-worktree contract (D-02, D-03, D-08, D-10) at
the model level so the heavy `window.py` wiring in plans 02/03 builds against a
tested, stable shape — and so the GTK-free invariant is provably preserved.
Output: Extended `src/arduis/session.py` + updated/added tests in `tests/test_session.py`.
</objective>

Current public surface (src/arduis/session.py):
```python
AGENT_FEED: bytes = b"claude\n"   # D-08 — bytes literal, do NOT change

class SessionState(str, Enum):
    ACTIVE = "active"
    HIBERNATED = "hibernated"

@dataclass
class WorktreeSession:
    session_id: str
    branch: str
    worktree_dir: str
    repo_root: str
    state: SessionState = SessionState.ACTIVE
    pid: int | None = None          # legacy single-terminal fields → fold into terminals[0]
    pgid: int | None = None
    rss_kb: int | None = None
    def to_dict(self) -> dict: ...   # asdict(self)

class SessionStore:
    def add(self, s) -> None
    def get(self, sid) -> WorktreeSession | None
    def by_branch(self, branch) -> WorktreeSession | None
    def all(self) -> list[WorktreeSession]
    def to_list(self) -> list[dict]

def hibernate_fields(session) -> None   # flips state, clears pid/pgid, keeps dir
```

Recommended target shape (from RESEARCH.md Pattern 2):
```python
@dataclass
class TerminalRecord:
    term_id: str           # stable leaf key, e.g. "feat:t0"
    kind: str              # "agent" (claude fed) | "shell" (plain zsh)
    pid: int | None = None
    pgid: int | None = None
    rss_kb: int | None = None
```

| T-03.1-02 | DoS (resource leak) | hibernate_fields clearing pgids | mitigate | hibernate_fields clears EVERY terminal's pgid so no group is forgotten by the caller's teardown loop (D-08, Pitfall 3); the actual os.killpg lives in window.py (plan 03). |
