# Phase 03.1 Plan 02: Per-Worktree Workspace Swap Summary

**Pivoted window.py from one global LayoutModel-of-worktrees to one LayoutModel PER worktree-of-terminals: the canvas shows exactly one worktree's terminals, sidebar row activation swaps the whole workspace (rebuild-on-switch, live agents preserved), and a new worktree is born eager with 2 terminals — left agent (claude-fed) + right plain shell.**

## Accomplishments

- **Per-worktree LayoutModel map** — `self._layout = LayoutModel()` became `self._layouts: dict[str, LayoutModel]` + `self._active_workspace_sid`, with `_workspace_layout(sid)` (lazy create) and `_active_layout()` helpers. Widget maps (`_leaf_by_sid`/`_term_by_sid`) re-keyed from worktree sid to terminal id (`"main:t0"`, `"feat:t0"`, `"feat:t1"`).
- **Generic reflect path** — `_reflect_layout` reads `self._active_layout()` (not a global tree), preserving the unparent-before-rehang dance (Pitfall 1). The pinned main row renders through the same path as any worktree — no `_MAIN_SID` special-case branch.
- **Whole-workspace swap** — `_swap_workspace(sid)` sets the active sid, reflects, and focuses that worktree's focused terminal. `_on_row_activated` calls it for every row (including main, D-07). The obsolete Phase-3 `resolve_selection`/`_focus_or_swap_session`/`_rebind_leaf` pane focus-or-swap path is gone.
- **Eager 2-terminal worktree create** — `_open_and_add` builds the worktree's own LayoutModel (`model.split(agent_tid, shell_tid, "h")` — left agent, right shell), creates both terminals via `_make_terminal()`, makes the new worktree the visible workspace, and after `git worktree add` succeeds spawns BOTH eagerly: agent fed `AGENT_FEED` (`kind == "agent"` guard), shell plain (D-02/D-03).
- **N-terminal spawn + RAM + teardown** — `_spawn_into`/`_make_wt_spawn_cb` take `term_id`+`kind` and write pid/pgid onto the matching `TerminalRecord`; RAM poll/footer sum every terminal's process group (D-10); `_split_active_pane`/`_next_term_id` let later splits spawn agent terminals into the active worktree.

## Decisions Made

- The pinned main workspace is a regular 1-terminal workspace routed through the same swap/reflect path — no `_MAIN_SID` special-casing (RESEARCH.md anti-pattern avoided).
- Workspace swap is rebuild-on-switch (RESEARCH.md Pattern 3): VTE widgets and their PTY children survive unparenting, so swapping away and back preserves splits/zoom/focus and the live agent — terminals are never respawned on swap.
- A worktree's RAM is the SUM of all its terminal process groups (D-10), and every terminal group is torn down on hibernate/close so split agents are never orphaned.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] `_on_close_request` referenced the removed `session.pid` field**
- **Found during:** Task 3 (continuation review of the uncommitted diff)
- **Issue:** Plan 01 folded the single `pid`/`pgid`/`rss_kb` fields into `terminals[0]` and removed them from `WorktreeSession`. The app-exit teardown loop still did `if session.pid: self._teardown_pgid(session.pid)`, which would raise `AttributeError` on window close (and, even if it existed, would orphan every split/shell terminal group).
- **Fix:** Routed the loop through the already-introduced `_teardown_session_terminals(session)` so EVERY terminal's process group is killed (Pitfall 3/4 — no orphans).
- **Files modified:** `src/arduis/window.py` (`_on_close_request`)
- **Verification:** `grep` confirms no `session.pid`/`session.pgid`/`session.rss_kb` code references remain; full pytest suite (53) green; file parses.
