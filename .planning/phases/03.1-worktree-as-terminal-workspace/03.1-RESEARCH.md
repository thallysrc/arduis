# Phase 03.1: worktree-as-terminal-workspace - Research

**Researched:** 2026-06-10
**Domain:** GTK4/libadwaita + VTE widget-tree refactor of an existing codebase (model pivot, not new tech)
**Confidence:** HIGH

## Summary

This is a **refactor/pivot phase**, not a greenfield one. There is no new library to adopt and no
external dependency to verify — the entire job is restructuring the *existing* arduis codebase to
match the approved 3-level interface model in `docs/MOTIVATION.md`. Phase 03 built a model where each
GTK pane shows a *different worktree* (one global `LayoutModel`, leaves keyed by worktree
`session_id`). Phase 03.1 inverts that: **the canvas shows the terminals of ONE worktree at a time**,
each worktree owns its own layout tree, and selecting a sidebar row swaps the entire workspace
(tmux mental model: panes = terminals, windows = worktrees).

The good news, confirmed by reading the source, is that the GTK-free `layout.py` (227 lines) is
**directly reusable as-is** — it is a generic binary tree of string-keyed leaves; it does not care
whether a leaf id means "a worktree" or "a terminal". The pivot is overwhelmingly concentrated in
`window.py` (1246 lines), which holds every integration point CONTEXT.md flags. The two hardest
correctness risks are both already-known GTK4 hazards visible in the current code: the
**single-parent reparenting rule** (you must `unparent()` a VTE widget before re-hanging it, or GTK
aborts) and **VTE process teardown** (every terminal's process group must be SIGHUP→SIGKILL'd on
hibernate/close — "no orphans" is the bar). The central discretion call is *how to swap the visible
canvas*: rebuild-on-switch (reuse the existing `_reflect_layout` unparent dance) vs. a `Gtk.Stack`
of per-worktree canvases. Both are viable; rebuild-on-switch is lower-risk because it reuses a code
path that already handles the single-parent rule and keeps exactly one widget tree live.

**Primary recommendation:** Keep `layout.py` and `session.py` GTK-free and serializable; move from
*one global `LayoutModel`* to *one `LayoutModel` per worktree* keyed on the session; re-target the
existing prefix actions (D-06) without touching the keymap dispatcher; and reuse the existing
`unparent()`-before-rehang reflection and `_teardown_pgid` group-kill paths so the two GTK hazards
stay solved. Model N terminals per worktree as a child "terminal" record list on the session (keeps
the store serializable and lets each terminal carry its own pgid for RAM summing and teardown).

**Scope**

- **D-07:** The pinned `main · zsh` row becomes a **regular workspace** with 1 terminal (the $HOME

- **D-08:** **Hibernate kills ALL the worktree's terminals** (every process group: agent + shells,

- **D-11:** The active-agents **cap counts active (non-hibernated) WORKTREES**, not terminals.

- **No-orphan teardown.** `os.killpg` the child's process GROUP (SIGHUP, then SIGKILL after the

- **GTK enforcement:** all file edits go through a GSD workflow (this phase is one).

## Architecture Patterns

### Current structure (Phase 03 — being pivoted)
```
ArduisWindow
├── _store: SessionStore               # all worktrees
├── _layout: LayoutModel               # ONE global tree; leaves keyed by worktree sid
├── _leaf_by_sid / _term_by_sid        # one terminal per worktree
└── _canvas_slot                       # reflects the single global tree
```

### Target structure (Phase 03.1)
```
ArduisWindow
├── _store: SessionStore               # all worktrees (extended: N terminals each)
├── per-worktree LayoutModel           # one tree PER worktree, keyed on session id
│   └── leaves keyed by TERMINAL id    # leaf = a terminal, not a worktree
├── _active_workspace_sid              # which worktree's canvas is visible
├── widget maps keyed by terminal id   # leaf/terminal widgets per terminal
└── _canvas_slot                       # reflects ONLY the active worktree's tree
```

### Pattern 1: One `LayoutModel` per worktree (D-04)
**What:** Replace the single `self._layout = LayoutModel()` with a per-worktree mapping
(`dict[worktree_sid, LayoutModel]`), plus an `_active_workspace_sid` pointer. Leaves now key on a
**terminal id** (e.g. `f"{worktree_sid}:t0"`, `:t1`, …), not a worktree id.
**When to use:** Every worktree (including the pinned `main` row, D-07).
**Why it works:** `layout.py` is fully id-agnostic — `split`, `close_leaf`, `zoom`, `preset`,
`touch`, `resolve_selection` operate on opaque string keys. [VERIFIED: read `layout.py` — no
worktree-specific logic anywhere]
**Note:** `preset`/`grid2x2`/`columns` and the `⌥ Layout` menu now operate within a single
workspace's terminals, which is more natural than the Phase-3 cross-worktree presets.

### Pattern 2: N terminals per worktree on the store (Discretion)
**What:** Extend `WorktreeSession` to hold a list of terminal records (recommended) rather than a
single `pid`/`pgid`. Keep it GTK-free and serializable.
**Recommended shape:**
```python
@dataclass
class TerminalRecord:
    term_id: str          # stable leaf key, e.g. "feat:t0"
    kind: str             # "agent" (claude fed) | "shell" (plain zsh)
    pid: int | None = None
    pgid: int | None = None
    rss_kb: int | None = None

@dataclass
class WorktreeSession:
    session_id: str
    branch: str
    worktree_dir: str
    repo_root: str
    state: SessionState = SessionState.ACTIVE
    terminals: list[TerminalRecord] = field(default_factory=list)
    # legacy pid/pgid/rss_kb fields: migrate into terminals[0]
```
**Why a record list over a bare pgid list:** RAM summing (D-10) and teardown (D-08) both iterate
pgids; the `kind` field also encodes which terminals re-feed `claude` on resume (D-09) — the default
layout is "one agent + one shell". A bare `list[int]` loses the agent/shell distinction.
**Serializability:** `dataclasses.asdict` recurses into nested dataclasses and lists, so
`store.to_list()` + `json.dumps` still works — but **add a test** mirroring `test_store_serializable`
to lock it. [VERIFIED: `asdict` recursion is standard stdlib behavior]
**Migration mechanics (Discretion):** Phase 2/3 stored a single `pid`/`pgid`/`rss_kb` on the session.
Cleanest path: fold those three fields into `terminals[0]` and drop them from the session (in-memory
only — D-13 says serializable, not persisted, so there is **no on-disk migration to do**). Update the
existing `test_session.py` `test_hibernate_model` / `test_store_serializable` to the new shape.

### Pattern 3: Workspace swap (the central Discretion call)
**What:** Selecting a sidebar row swaps the *entire visible canvas* to that worktree's `LayoutModel`.

Two viable implementations:

| Approach | How | Pros | Cons |
|----------|-----|------|------|
| **Rebuild-on-switch (RECOMMENDED)** | Set `_active_workspace_sid = sid`; call `_reflect_layout()` which reads the active worktree's tree | Reuses the existing `unparent()`-before-rehang path (Pitfall 1 already solved); only ONE widget tree live at a time; minimal new code | Must keep per-terminal leaf/term widget maps so widgets survive being detached from the slot |
| **`Gtk.Stack` of canvases** | One child canvas per worktree added to a `Gtk.Stack`; `set_visible_child` on swap | No reparenting on swap; VTE keeps rendering in the background | More live widget trees = more VTE render surfaces resident; orphan risk if a worktree is removed without removing its stack page; background VTE keeps painting (minor cost) |

**Recommendation: rebuild-on-switch.** The existing `_reflect_layout` already correctly detaches
every mapped leaf (`leaf.unparent()`) before re-hanging it, which is exactly the single-parent
discipline a swap needs. Crucially, **VTE widgets and their PTY child processes survive being
unparented** — detaching a `Vte.Terminal` from its container does NOT kill the shell; the process
keeps running, so switching away and back preserves the live agent. Keep `_leaf_by_sid` /
`_term_by_sid` keyed by **terminal id** so the widgets persist across swaps and only the *visible*
subset is parented into `_canvas_slot`. This satisfies "preserves splits, zoom, and pane focus"
(D-04) because each worktree's `LayoutModel` retains its own `root`/`focused_id`/`_pre_zoom`/`_mru`.
[VERIFIED: read `_reflect_layout` — the unparent dance is already present and correct]

### Pattern 4: Re-targeted prefix actions (D-06)
**What:** Keep `keymap.py` and the `_on_key` capture-phase state machine **byte-for-byte**. Only
`_run_action`'s handlers change targets:

| Key | Phase 03 target | Phase 03.1 target |
|-----|-----------------|-------------------|
| `h/j/k/l` (`focus_dir`) | neighbor leaf across the global tree (= neighbor worktree) | neighbor **terminal** in the active worktree's tree |
| `n/p` (`worktree`) | cycle sidebar selection + focus-or-swap | switch the whole **workspace** (already sidebar cycle — semantics now "swap workspace") |
| `<1-9>` (`jump`) | Nth sidebar row | Nth sidebar row (worktree) — unchanged ordering (Discretion: keep sidebar row order) |

`_focus_neighbor` switches from `self._layout.visible_ids()` to
`self._workspace_layout(active).visible_ids()`. `_cycle_worktree` / `_jump_to_row` already drive the
sidebar; they now trigger a *workspace swap* instead of a focus-or-swap-into-pane. The keymap
dispatcher and prefix arming logic are untouched (Pitfall 6 handling stays valid). [VERIFIED: read
`keymap.py` + `_on_key` + `_run_action`]

### Pattern 5: Per-worktree RAM = sum of N terminal pgids (D-10)
**What:** `_poll_ram` currently sums one `session.pgid` per worktree. Change it to iterate
`session.terminals` and sum `resource_monitor.group_rss_kb(t.pgid)` for each non-None pgid. The row
sub-line shows the worktree total; the footer aggregate sums across worktrees (already does, with the
new per-worktree totals). `resource_monitor.py` itself is **unchanged**. [VERIFIED: read
`resource_monitor.py` + `_poll_ram`]

### Anti-Patterns to Avoid
- **Re-hanging a VTE widget without `unparent()` first.** GTK4 enforces single-parent; a widget that
  still has a parent will abort with "widget already has a parent". The existing `_reflect_layout`
  unparents first — preserve that ordering for the new per-terminal maps.
- **Keying layout leaves by worktree id in 03.1.** Leaves are now terminals. Reusing worktree ids as
  leaf ids will collide the moment a worktree has 2 terminals.
- **Special-casing the `main` row.** D-07 makes it a regular 1-terminal workspace. Adding bespoke
  `if sid == _MAIN_SID` branches in the swap path is the thing D-07 explicitly forbids.
- **Letting the cap count terminals.** D-11: the cap counts active *worktrees*. `caps.active_count`
  already counts ACTIVE sessions — do not change it to count terminals.
- **Lazy-spawning the 2nd terminal.** D-03 says spawn it eagerly with the workspace.

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Binary split/zoom/preset/MRU tree | A new per-terminal tree structure | The existing `layout.py`, instantiated per worktree | It's already id-agnostic and fully tested (`test_layout.py`) |
| Process-group RSS accounting | A new `/proc` walker for N terminals | `resource_monitor.group_rss_kb` per pgid, summed | Already handles pid-reuse/race/permission pitfalls |

## Common Pitfalls

### Pitfall 1: Reparenting a VTE widget that still has a parent
**What goes wrong:** GTK4 aborts the process ("widget already has a parent") when you `set_child` /
`set_start_child` a widget already parented elsewhere.
**Why it happens:** A workspace swap moves the same VTE widgets in and out of `_canvas_slot`.
**How to avoid:** Reuse the existing `_reflect_layout` discipline — call `leaf.unparent()` on every
mapped leaf BEFORE building the new tree, then re-hang only the active workspace's leaves.
**Warning signs:** Hard crash (not a Python exception) the second time you switch workspaces.

### Pitfall 2: Killing the PTY child by unparenting the terminal
**What goes wrong:** Assuming you must respawn a terminal when switching workspaces.
**Why it happens:** Misreading "swap the canvas" as "destroy and recreate the canvas".
**How to avoid:** Unparenting a `Vte.Terminal` does NOT kill its child process — keep the widget
alive in `_term_by_sid` and only detach/reattach it. Respawn happens only on resume-from-hibernate
(D-09), never on a plain workspace switch.
**Warning signs:** Agents restart / lose scrollback when you switch away and back (violates D-04).

### Pitfall 3: Hibernate leaving orphaned terminal groups
**What goes wrong:** Hibernate kills only `terminals[0]` and leaves the split agents running → RAM
not actually freed (violates D-08).
**Why it happens:** Phase-2/3 `_on_hibernate` kills a single `session.pid`.
**How to avoid:** Iterate ALL `session.terminals`, `_teardown_pgid(t.pid)` each, then
`hibernate_fields`. After hibernate, clear the worktree's `LayoutModel` (D-09: resume rebuilds the
default layout, not the saved tree).
**Warning signs:** Footer RAM total doesn't drop to zero for the hibernated worktree; `ps` shows
surviving `claude`/`zsh` under the worktree dir.

### Pitfall 4: Window-close teardown missing the new terminals
**What goes wrong:** `_on_close_request` tears down `self._shell_pid` + one pid per session →
split-terminal agents orphan on app exit.
**Why it happens:** Same single-pid assumption.
**How to avoid:** Iterate every terminal of every session (and the main row's terminal) in the
close handler. "No orphans" is a hard acceptance bar (CLAUDE.md).
**Warning signs:** Orphaned `claude` processes after closing arduis.

### Pitfall 5: `feed_child` with a str
**What goes wrong:** `TypeError: Must be number, not str` at the 0.76 floor when feeding `claude` into
a new split (D-05).
**How to avoid:** Always feed `AGENT_FEED` (the `b"claude\n"` bytes literal). [VERIFIED: documented in
`session.py` + `test_session.py`]

### Pitfall 6: GTK-free modules accidentally importing gi
**What goes wrong:** Extending `session.py` with a terminal-aware helper and reaching for a GTK type.
**How to avoid:** Keep `TerminalRecord` a plain dataclass; the `test_session_module_is_gtk_free` /
`test_layout_is_gtk_free` assertions will fail loudly if `import gi` leaks in. [VERIFIED: tests exist]

## Code Examples

### Per-worktree layout lookup + swap (rebuild-on-switch)
```python
# window.py — replace the single self._layout with a per-worktree mapping.
self._layouts: dict[str, LayoutModel] = {}        # worktree_sid -> tree
self._active_workspace_sid: str | None = None

def _workspace_layout(self, sid: str) -> LayoutModel:
    return self._layouts.setdefault(sid, LayoutModel())

def _swap_workspace(self, sid: str) -> None:
    self._active_workspace_sid = sid
    self._reflect_layout()                         # reads the active tree
    # focus the active workspace's focused terminal
    model = self._workspace_layout(sid)
    term = self._term_by_sid.get(model.focused_id)
    if term is not None:
        term.grab_focus()
```

### Reflection reads the ACTIVE workspace's tree (minimal delta to existing code)
```python
# _reflect_layout: only the root source changes; the unparent dance stays.
def _reflect_layout(self) -> None:
    if self._canvas_slot.get_child() is not None:
        self._canvas_slot.set_child(None)
    for leaf in self._leaf_by_sid.values():        # leaf_by_sid now keyed by TERMINAL id
        if leaf.get_parent() is not None:
            leaf.unparent()
    model = self._workspace_layout(self._active_workspace_sid)
    self._canvas_slot.set_child(self._build_widget(model.root))
    # focus ring on model.focused_id ...
```

### Per-worktree RAM sum (D-10)
```python
def _poll_ram(self) -> bool:
    for session in self._store.all():
        if session.state != SessionState.ACTIVE:
            continue
        total = 0
        for t in session.terminals:
            if t.pgid is None:
                continue
            t.rss_kb = resource_monitor.group_rss_kb(t.pgid)
            total += t.rss_kb
        label = self._subline_by_sid.get(session.session_id)
        if label is not None:
            label.set_text(f"claude · {resource_monitor.format_ram_kb(total or None)}")
    self._update_footer()
    return GLib.SOURCE_CONTINUE
```
