# Phase 03.1 Plan 01: N-Terminal Session Model Summary

Extended the GTK-free `WorktreeSession` so each worktree owns a LIST of `TerminalRecord`s (default 2 ‚Äî one `agent`, one `shell`) instead of a single pid/pgid, with `hibernate_fields` now clearing every terminal's pgid and the store still JSON-serializable via `dataclasses.asdict`.

## What Was Built

- **`TerminalRecord`** dataclass: `term_id` (stable leaf key, e.g. `"feat:t0"`), `kind` (`"agent"` | `"shell"`), `pid`, `pgid`, `rss_kb` ‚Äî the per-terminal teardown handle and RAM datum.
- **`default_terminals(sid)`** module function: returns `[TerminalRecord(f"{sid}:t0", "agent"), TerminalRecord(f"{sid}:t1", "shell")]` ‚Äî the 2-terminal workspace default (D-02/D-03).
- **`WorktreeSession.terminals`**: `list[TerminalRecord]` with `field(default_factory=list)`. The legacy standalone `pid`/`pgid`/`rss_kb` fields were removed ‚Äî that data now lives in `terminals[0]`.
- **`hibernate_fields`**: rewritten to flip state to `HIBERNATED` and clear `pid`/`pgid` on EVERY terminal (Pitfall 3 ‚Äî no group forgotten so RAM can't leak), keeping `worktree_dir`/`repo_root` on disk and each terminal's `rss_kb` frozen.
- **`to_dict`** stays `asdict(self)` ‚Äî it recurses into the terminals list (A2), so the store remains JSON-serializable.
