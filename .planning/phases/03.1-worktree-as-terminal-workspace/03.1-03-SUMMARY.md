# Phase 03.1 Plan 03: Lifecycle, Keybinding & RAM Re-target Summary

**Re-targeted the C-Space prefix onto the per-worktree-terminals model (h/j/k/l step terminals, n/p + number switch whole workspaces incl. main), summed RAM across each worktree's terminal process groups, made hibernate kill every terminal group and resume rebuild the default 2-terminal layout, and root-caused + fixed a nested-Gtk.Paned split collapse with a max-position proportional positioner — all confirmed via a manual UAT (A1: live agents survive workspace swap) after two fix rounds.**

- **Duration:** multi-session (Tasks 1-2 autonomous; Task 3 manual UAT failed once → 2 fix rounds incl. a headless broadway debug session → approved)
- **Completed:** 2026-06-10
- **Tasks:** 3 (2 auto + 1 blocking human-verify checkpoint)
- **Files modified:** 2 (`src/arduis/window.py`, `tests/test_layout.py`)

## Accomplishments

- **Keybinding re-target (D-06/D-07)** — `_focus_neighbor` now sources from `self._active_layout().visible_ids()`, so C-Space h/k/j/l moves focus between the TERMINALS of the visible worktree (tree-order). `_cycle_worktree` (n/p) and `_jump_to_row` (<number>) route through `_all_workspace_rows` + `_on_row_activated`, switching the WHOLE workspace and reaching the pinned main row (1-indexed, main = row 1). The keymap dispatcher and prefix arming are untouched.
- **Per-worktree RAM (D-10 / RAM-03)** — `_poll_ram` iterates `session.terminals`, writes each `t.rss_kb = group_rss_kb(t.pgid)`, and sets the row sub-line to the SUM; `_update_footer` totals every active worktree's terminal RSS while the active COUNT stays `caps.active_count` (worktrees, not terminals — D-11 / RAM-02).
- **Hibernate-all + resume-default (D-08/D-09)** — `_on_hibernate` tears down every terminal group via `_teardown_pgid`, drops the saved LayoutModel (`self._layouts.pop(...)`), clears that worktree's widget-map entries, and swaps to main if the hibernated worktree was visible. `_on_resume` rebuilds the DEFAULT 2-terminal layout (`default_terminals` → fresh LayoutModel, agent + shell split "h", agent fed `AGENT_FEED` only); extra splits are not restored.
- **No-orphan close (Pitfall 4)** — `_on_close_request` tears down the main shell AND every `t.pid` of every session through the existing SIGHUP→SIGKILL grace path.
- **Canvas-reflection repair (UAT round 1)** — new `_close_terminal` kills the pgid, cleans the widget maps, re-reflects, and falls back to main on the last close; `_focus_leaf` was rewritten to update the focus ring only (no paned-tree rebuild); sidebar selection is kept in sync with the visible workspace.
- **Nested-Paned split fix (UAT round 2, headless-debugged)** — `_init_paned_position` was rewritten from one-shot map-time `get_width()//2` (which pinned nested paneds to a ~14px transient allocation, collapsing a terminal to a ~6px left sliver) to a `notify::max-position` proportional positioner with `notify::position` drag-ratio learning. Verified headless: 3 terminals at 281/281/351px, close re-fills at 350/349px, zero GTK-CRITICAL on the reflect path.
- **Dead-UI removal (UAT)** — removed the pre-pivot Layout button/menu and the false "C-Space z zoom" hint, so the UI no longer advertises actions that don't exist after the pivot.

## Decisions Made

- **Keymap frozen, handlers re-targeted** — the C-Space prefix state machine in `keymap.py` is reused byte-for-byte; only `window.py` action TARGETS change. No new keymap entries (zoom `z` / split chords stay button-only, per RESEARCH Open Question 1).
- **n/p + number reach main** — the cycle/jump order includes the pinned main row (1-indexed) so every workspace is keyboard-reachable (D-07 / Discretion A3).
- **Resume is a default rebuild, not a tree restore** — hibernate drops the LayoutModel; resume deterministically rebuilds the agent+shell default (D-09). Falling back to main when hibernating the visible worktree resolves RESEARCH Open Question 2.
- **Nested-Paned sizing via max-position** — root-caused the split collapse to premature one-shot map-time positioning; the fix uses the real usable extent (`max-position`) and learns the user's drag ratio, settling correctly even for nested paneds.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Canvas reflection dropped/crushed terminals on split, close, and focus**
- **Found during:** Task 3 (manual UAT, round 1 — 3-split collapse, blank canvas after pane close, focus rebuilding the tree)
- **Issue:** Splitting to 3 terminals collapsed the layout; closing a pane left a blank canvas; `_focus_leaf` rebuilt the paned tree on every focus move (churning widgets). A dead pre-pivot "Layout" button and a false "C-Space z zoom" hint were also still present post-pivot.
- **Fix:** Added `_close_terminal` (kill pgid → clean maps → re-reflect → fall back to main on last close); rewrote `_focus_leaf` to update only the focus ring (no tree rebuild); kept sidebar selection in sync with the visible workspace; removed the dead Layout button/menu and the false zoom hint.
- **Files modified:** `src/arduis/window.py`
- **Verification:** Re-ran the UAT (round 1 issues 2-4 cleared); added layout invariant tests (`68fa101`).
- **Committed in:** `b487dfe` (UAT fix round 1), `68fa101` (invariant tests)

**2. [Rule 1 - Bug] Nested Gtk.Paned splits collapsed to a ~6px sliver (3+ terminals)**
- **Found during:** Task 3 (manual UAT — split collapse persisted after round 1)
- **Issue:** `_init_paned_position` pinned each paned's split to `get_width()//2` on its FIRST `map` signal then disconnected (one-shot). For NESTED paneds the map fires while the widget still has a tiny transient allocation (~28px → pos 14), so the outer split pinned to ~14px and crushed the inner subtree to a ~6px left sliver, never correcting.
- **Fix:** Replaced the one-shot map/`get_width()` positioning with a `notify::max-position` proportional handler (`position = int(max_position * ratio)`, ratio default 0.5, re-applied on every max-position change) plus a `notify::position` drag-ratio learner guarded by an `_applying` flag. `max-position` reflects the real usable extent and settles correctly for nested paneds.
- **Files modified:** `src/arduis/window.py`
- **Verification:** Headless broadway driver after the fix — 3 terminals at outer pos 352/710 (50%), inner pos 173/352 (50%), VTE widths 281/281/351px (all visible); close 1-of-3 re-fills at 350/349px; reflect path emits zero GTK-CRITICAL. Full debug session recorded in `.planning/debug/paned-collapse-3-terminals.md`.
- **Committed in:** `65a3989` (UAT fix round 2, root-cause fix)
