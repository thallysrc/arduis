Headless harness pattern (Phase 4 precedent + dev-environment memory):
- Start `gtk4-broadwayd :9N` (pick a free display number, e.g. :95) in the background.
- Run the harness with `GDK_BACKEND=broadway BROADWAY_DISPLAY=:9N` in its env.
- DO NOT override XDG_RUNTIME_DIR â€” broadway finds its socket there (the Phase-4 smoke lesson).
- Kill the broadwayd (and any spawned process groups) in a finally block (no orphans).
- Construct the Adw app/window, iterate the GLib loop a few times for spawns/CSS to settle.

Seams the smoke inspects (from 05-03):
```python

win._agent_config.command               # the configured agent command
```
