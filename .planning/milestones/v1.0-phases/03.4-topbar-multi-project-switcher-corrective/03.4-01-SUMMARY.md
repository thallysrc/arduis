## What Was Built

- **`Project` dataclass** (project.py, GTK-free): `root` + `member_repos` + its OWN `SessionStore`
  (D-02 â€” never shared) + `last_active_task` switch-back pointer + per-project `compose_path` /
  `container_state`, with a `name` property = basename. Replaces window.py's one-project singletons
  at the model level (D-01).
- **`ProjectRegistry`** (project.py): open projects keyed by absolute root with one active pointer;
  `add` (dedup-replace), `get`, `all` (insertion order), `active`, `set_active`, `remove` (clears
  the active pointer iff it pointed at the removed project). In-memory only â€” teardown is window.py's job.
- **`ensure_project(registry, root, members)`** (D-07): idempotent launch-autoregister â€” returns the
  existing instance for a known root (no duplicate, members untouched), else builds + adds a fresh Project.
- **`project_term_id(root, term_id)`** (A3 resolved): prefixes an 8-hex `sha1(root)` discriminator
  so two projects with same-named branches (`feat:t0`) never collide on the global attention
  status-file path; the result stays inside `attention.sanitize_term_id`'s allowlist (hex + `:` only).
- **`projects_store.py`** (GTK-free, stdlib `json`): `save_projects` (atomic mkstemp-in-dir â†’ `os.replace`,
  best-effort, mid-write failure unlinks temp + leaves original intact â€” T-03.4-01) and `load_projects`
  (tolerant: skips missing roots per D-06, drops invalid `last_active`, bad-json/not-a-dict/missing â†’
  `([], None)` â€” T-03.4-02). Path is an argument so the module imports no gi.
- **PROJ-01 requirement** registered in REQUIREMENTS.md (new Projetos subsection + traceability row
  `Phase 03.4 | Pending` + coverage 32â†’33, Unmapped still 0).
