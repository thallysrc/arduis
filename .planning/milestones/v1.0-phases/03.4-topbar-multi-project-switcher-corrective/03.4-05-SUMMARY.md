# Phase 03.4 Plan 05: Lifecycle Acceptance Smoke + UAT Gate Summary

Built the headless acceptance smoke for the persistence + lifecycle wiring and confirmed the full
suite + surviving smokes are green before the live human checkpoint. The smoke proves — without a
display or any manual interaction — that projects.json round-trips across a simulated restart
(SC-2/D-05/D-07), that a deleted remembered root is skipped on load (D-06), and that removing a
project with no live tasks drops it silently and rewrites the JSON (D-10). The live process-tree
guarantees (both-alive D-08, remove-teardown D-10, no-orphans-on-exit D-11) remain the human UAT —
broadway can't host real agents/docker, exactly as the plan scoped.

## What Was Built

- **Task 1 — headless lifecycle smoke (`tests/smoke/test_project_lifecycle_smoke.py`, 197 lines).**
  Sandbox HOME + isolated XDG_CONFIG_HOME (projects.json lands in the sandbox, never the user's real
  `~/.config/arduis` — T-03.4-16), /tmp multi-repo git fixtures for two roots (Livon-Saude shape +
  KarveLabs shape). Three groups, 9 checks, all PASS:
  - **(a) PERSIST across restart (SC-2/D-05/D-07):** launch INSIDE root A → A auto-registers;
    `_open_project(rootB)` → projects.json lists A+B; destroy the window; build a FRESH window from a
    NEUTRAL cwd in the same sandbox → `_init_projects` restores BOTH from disk and renders two tabs.
  - **(b) MISSING-ROOT SKIP (D-06):** a projects.json listing a never-created root + a real one →
    a new window loads only the real one, renders one tab, no crash.
  - **(c) REMOVE-NO-LIVE SILENT (D-10):** two registered projects, no live tasks →
    `_remove_project(rootB)` drops B from the registry, rewrites projects.json to A only, leaves one
    tab — no `Adw.AlertDialog` on the no-live path.
  - SMOKE_SKIP + `sys.exit(0)` before any window when `gtk4-broadwayd` is absent.

- **Task 2 — full suite + all smokes green before the checkpoint.** `pytest` = **426 passed**
  (3 pre-existing fork DeprecationWarnings in test_exit_decode.py, out of scope). Smokes:
  lifecycle 9/9, switch 5/5, setup-feed 7/7. Two in-scope fixes were required to clear the

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Stale hardcoded REPO path in test_project_switch_smoke.py**
- **Found during:** Task 2 (running every smoke).
- **Issue:** the Plan-03 switch smoke hardcoded `REPO = ".../agent-ae6fed5f8e71a3a43"` — a different
  (now non-existent) worktree — so `sys.path.insert` added a missing `src` dir and `import arduis`
  raised `ModuleNotFoundError`. The harness could never run outside that one worktree.
- **Fix:** `REPO = os.path.abspath(os.path.join(os.path.dirname(__file__), "..", ".."))` (the same
  worktree-portable pattern used in the new lifecycle smoke). Now PASS 5/5.
- **Commit:** 98dbf1c

- **Fix:** deleted the smoke. Member-repo selection moved to the New-task dialog (D-12, covered by
  unit tests); the topbar's actual new behavior is covered by the switch + lifecycle smokes.

## Threat Mitigations Applied

- **T-03.4-16** (Tampering — smoke must not touch real state): the lifecycle smoke sets both `HOME`
  and `XDG_CONFIG_HOME` to a tmpdir sandbox, so `projects.json` resolves inside the sandbox and the
  user's real `~/.config/arduis/projects.json` is never written. The sandbox is `rmtree`'d in finally.
- **T-03.4-15** (resource leak / no-orphans): deliberately NOT auto-approvable — the live
  process-tree `pgrep`/`docker ps` zero-orphans check is the blocking human UAT (Task 3), which a
  headless smoke cannot prove. Returned as a checkpoint, not self-signed.
