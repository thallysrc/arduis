<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| meson install → filesystem | install() places files in system prefixes at package build/install time |
| launcher → Python import | `bin/arduis` runs as the user; its import path must resolve only the installed package |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-04 | Elevation of privilege | generated `bin/arduis` launcher | mitigate | Launcher contains no `eval`/`exec` of external input, no writable-by-all path; installs `rwxr-xr-x` (no setuid); imports only the installed `arduis` package |
| T-09-05 | Tampering | meson install paths | mitigate | Use `get_option(...)` XDG dirs (no hardcoded world-writable paths); `install_sources` to purelib only; no network fetch during configure/install |
| T-09-06 | Spoofing | importlib.resources hook resolution | mitigate | Hook ships as package data via `preserve_path`; resolves from the installed package, not an arbitrary path (RESEARCH Pitfall 1) |
</threat_model>
