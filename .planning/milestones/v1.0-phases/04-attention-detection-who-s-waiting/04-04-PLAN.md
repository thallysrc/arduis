<plan_decisions>
- **Degraded mode never auto-suspends:** D-12 keys suspension to "all agents ready/idle" —
  degraded mode has no `ready` (D-13), and killing processes on a coarse activity-timeout
  signal risks exactly the Pitfall-6 SIGKILL-a-working-agent failure. Auto-suspend therefore
  requires hook-derived states. Documented limitation, UAT-revisitable.
- **Degraded thresholds:** activity within the last 30s → running; bell → waiting (sticky
  until the next contents-changed activity burst); no activity for idle_minutes → idle.
  Pane badge label in degraded mode shows "esperando?" (with the question mark — D-13
  "labeled differently", lower confidence) instead of the dot-only treatment.
- **Suspension notification:** always notify on auto-suspend (even focused — the app just
  killed processes on the user's behalf; that must never be silent).
- **calm_since semantics:** wall-clock when the task's aggregate ENTERED ready/idle/ended;
  reset to None whenever aggregate is running/waiting/None or the task is not ACTIVE.
- **Manual-resume flag clearing:** `task.auto_suspended` is cleared on EVERY resume after
  selecting the feed, so one `--continue` resume never leaks into later manual cycles.
</plan_decisions>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| status aggregate → process-group SIGHUP/SIGKILL | a status misread can kill a working agent |
| terminal output (BEL) → status/notification | any program in the PTY can ring the bell |
| resume feed bytes → the user's shell | bytes injected into a live zsh |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-18 | DoS (process kill) | auto-suspend killing a running/waiting agent | mitigate | Policy gate tested in Plan 02 (RUNNING/WAITING/None immune at any age); degraded mode excluded entirely; default OFF; reuses the proven no-orphan hibernate path (03.2-verified) |
| T-04-19 | Spoofing | any PTY program ringing BEL to fake "waiting" | accept | Degraded mode is explicitly lower-confidence (D-13) and labeled "esperando?"; worst case is a false hint, never a kill (no auto-suspend in degraded) |
| T-04-20 | Tampering | resume feed injection | mitigate | AGENT_RESUME_FEED is a bytes LITERAL constant (no interpolation of any user/task data into the fed line), mirroring the AGENT_FEED posture |
| T-04-21 | DoS | contents-changed event storm (TUI repaint) flooding the loop | mitigate | 1s throttle per terminal; handler is dict-write + occasional label flip |
| T-04-22 | Repudiation | silent background process killing | mitigate | Auto-suspend ALWAYS notifies + marks the row "suspensa" — the user can always tell the app did it |
</threat_model>
