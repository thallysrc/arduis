# Phase 03.2 Plan 02: window.py Project/Task Pivot Summary

**`window.py` migrated from `WorktreeSession` to `Task`: startup detects member repos (or the degenerate 1-repo project), the topbar shows the project name (no switcher), and "New task" picks 1+ repos + a branch to materialize a mirrored task folder (relative symlinks + chosen repos as worktrees via a per-repo chained `git worktree add`, never `--force`) opening a 2×N one-column-per-repo workspace.**

## Accomplishments
- **Project resolution (`_resolve_project`)** replaces `_resolve_repo_root`: `detect_member_repos(cwd)` (D-05 direct-subdir `.git` scan); if empty, the `git rev-parse --show-toplevel` fallback gives the degenerate 1-repo project (D-07, 03.1 behavior preserved). `_repo_root`/`_repo_name` are kept as project-root/name aliases for the pinned-leaf code.
- **`_member_repo_path`** maps a member NAME to its PATH: `project_root` in the degenerate case (sole member's path IS the root, not a subdir), else `<root>/<name>`.
- **Name-only topbar (D-06):** a stored `_title_widget` is updated to the project name (fallback "arduis"). No switcher.
- **New-task dialog (`_present_new_task_dialog`):** branch type-or-pick `Gtk.ComboBoxText` + a `Gtk.CheckButton` per member repo (all checked by default); requires ≥1 repo and a non-empty branch.
- **`_create_task`:** `makedirs` the task folder, materialize the mirror symlinks FIRST (`os.symlink(os.path.relpath(src, task_dir), dst)` guarded by `lexists`, best-effort), register the `Task` immediately (row appears), build the 2×N one-column-per-repo workspace (agent over shell within a column, columns joined horizontally), then kick the per-repo chain.
- **`_add_repo`:** per-repo chained `run_git_async` (born-HEAD → porcelain → branches → base → `resolve_repo_add` → `git worktree add`). An `("abort", reason)` (branch checked out elsewhere — never `--force`, D-13) is collected and the chain advances; non-zero adds are collected too; one consolidated error is surfaced at the end (D-10/OQ2).
- **Spawn-record migration:** `_spawn_into`/`_make_wt_spawn_cb` take a `Task` and find the `TerminalRecord` via `next((t for r in task.repos for t in r.terminals if t.term_id == term_id))`.
- **Lifecycle seams compile against `Task`:** `_rebuild_sidebar` (rows are Tasks, pinned row `<project> · zsh`), `_all_workspace_rows`, `_close_terminal`, `_split_active_pane`, `_poll_ram`, `_update_footer`, `_session_for_worktree_dir`, `_menu_session`, `_on_hibernate`, `_on_resume`, `_teardown_session_terminals` all iterate `task.repos[*].terminals`.

## Decisions Made
- **`_member_repo_path` degenerate mapping:** when the sole member name equals the project name and there is no `<root>/<name>` subdir, the member's path IS the project root (preserves the 03.1 single-repo behavior through the identical Task code path).
- **Per-repo independence:** each repo resolves and adds independently (best-effort, OQ2); a single abort or failed add never aborts the whole task and never deletes anything from disk (D-10).
- **`_on_resume` minimal re-target:** rebuilds the default 2×N per-repo layout so resume stays valid against `Task`; the full per-task RAM grouping, startup scan, and layout polish are explicitly Plan 03 (marked with a `# Plan 03` comment).
- **Removed `_abort_already_checked_out`:** the already-checked-out abort path is now produced inline by `resolve_repo_add` within `_add_repo`.
