# Phase 03.2: Projects and Cross-Repo Tasks — Verification Report

**Phase Goal:** Pivot level 1 of the interface from "topbar = single git repos" to project = multi-repo root folder. Introduce task as the unit of work: creating a task picks 1+ of the project's repos, creates a branch-named worktree in each, and materializes a task folder mirroring the root layout so the project's tooling runs verbatim inside the task. Sidebar pivots from worktrees to tasks. Single-repo project is the degenerate case. Hibernate/RAM/cap/teardown re-target from worktree to task.

**Verified:** 2026-06-12
**Status:** human_needed — 9/9 automated truths verified; 6 items require live GTK confirmation.
**Re-verification:** No — initial verification.

**User-decided deviations honoured (not flagged as gaps):**
1. Task workspace opens with ONE 2-terminal pair at the task-folder root regardless of repo count (supersedes D-01/D-02 per-repo columns — user decided live after a 6-repo project produced an unusable 2×6 grid).
2. Clicking a hibernated task navigates to an explicit "Retomar task" card; resume is cap-gated.
3. Clear pt-BR ref-namespace conflict message + zombie-task removal on all-repos-failed creation.
