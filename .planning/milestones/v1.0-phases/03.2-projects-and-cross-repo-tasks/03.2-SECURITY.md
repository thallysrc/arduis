# Phase 03.2 Security Audit — Projects and Cross-Repo Tasks

Verification of the threat registers declared in `03.2-01-PLAN.md`, `03.2-02-PLAN.md`,
and `03.2-03-PLAN.md` against the implemented code (`src/arduis/project.py`,
`task_layout.py`, `session.py`, `window.py`) and tests. The full suite passes
(88 tests, `/tmp/arduis-venv-ab12/bin/python -m pytest tests/ -q`).

> **Scope note (UX pivot):** late in the phase a user-decided pivot replaced the
> per-repo terminal columns with ONE task-level 2-terminal pair at the task-folder
> root. Threats referencing per-repo terminals were evaluated against the CURRENT
> code: `_all_task_terminals` (window.py:1564) iterates the task-level pair PLUS
> any per-repo splits, and `hibernate_fields` (session.py:182) clears BOTH
> `task.terminals` and `task.repos[*].terminals` — so the pivot did not narrow
> teardown/RAM coverage.

## Trust Boundaries Audited

| Boundary | Where crossed |
|----------|---------------|
| user-typed branch name → filesystem path | `task_dir_for` leaf, task folder + worktree dirs |
| user-typed branch name → git argv | `resolve_repo_add` → `worktree.py` argv builders |
| on-disk dir names (scanned) → git argv / Task model | `detect_member_repos`, `_scan_tasks`/`_dir_is_task` |
| symlink materialization | `_create_task` consuming `symlink_plan` pairs |
| process groups (pgid) → `os.killpg` | `_teardown_pgid` / `_teardown_session_terminals` |

## Accepted Risks Log

| Threat ID | Risk | Rationale | Residual exposure | Reviewed |
|-----------|------|-----------|-------------------|----------|
| T-03.2-08 | A partially-failed task creation leaves a half-materialized `../<root>-tasks/<branch>/` folder (symlinks and/or a subset of worktrees) on disk. | D-10 ("arduis never deletes from disk") is a deliberate, phase-wide product decision: auto-rollback would require arduis to delete user-adjacent directories, a worse failure mode than leftover folders.
