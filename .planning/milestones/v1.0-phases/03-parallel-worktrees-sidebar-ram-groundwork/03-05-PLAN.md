<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| keystroke capture → terminal vs. app | A window-level capture-phase controller sees every key before the focused `Vte.Terminal`; mis-handling could swallow user input or leak the prefix into the shell. |
| kernel `/proc` → ~2s poll on the GTK loop | The poll reads `/proc` for the user's own process groups on the main loop; a slow/unbounded read could stall the UI. |
| +New flow → active-agent cap | The cap is the only gate bounding resource use; a bypass defeats the lightweight promise. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-12 | Tampering (input theft) | capture-phase prefix controller | mitigate | `_on_key` returns True ONLY for the prefix keystroke while disarmed and for a recognized action key while armed; every other key returns False so the focused terminal receives all normal typing (Pitfall 6, verified in manual check 5). |
| T-03-13 | Denial of Service | ~2s `/proc` poll on the main loop | mitigate | Work bounded to the 5–12 active groups (~3 tiny reads/pid, sub-ms — Assumption A1); hibernated/pgid-None sessions skipped; per-pid errors swallowed; poll source removed on window close (no leaked source). Documented escalation to a `Gio.Subprocess` helper if jank appears (manual check 6). |
| T-03-14 | Elevation of Privilege (cap bypass) | +New create flow | mitigate | The flow routes through `caps.at_cap(self._store.all())` BEFORE add/spawn; at cap it blocks and forces a hibernate before proceeding — no path creates an agent past the cap (D-16, manual check 7). |
| T-03-15 | Information Disclosure | RSS shown for unrelated pids | mitigate | `group_rss_kb` sums only pids whose pgrp == the session's own `pgid` (a group arduis spawned); other users'/processes' pids are excluded and their `/proc` PermissionErrors are swallowed (handled in resource_monitor, Plan 03-03). |

Spoofing/Repudiation are N/A — local single-user desktop app, no auth/network surface.
</threat_model>
