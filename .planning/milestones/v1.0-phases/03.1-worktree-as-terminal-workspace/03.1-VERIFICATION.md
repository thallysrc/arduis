# Phase 03.1: worktree-as-terminal-workspace Verification Report

**Phase Goal:** Pivot the workspace model to the approved 3-level interface: the pane canvas shows the terminals of ONE worktree at a time (workspace), not multiple worktrees side by side. Each worktree owns its own layout tree and 2 default terminals (agent + shell); selecting a sidebar row swaps the whole workspace (tmux: panes = terminals, windows = worktrees). Re-targets Phase-3's PAR/LAYOUT/RAM behaviors under the new semantics; topbar/multi-repo (level 1) is out of scope.

### Deliberate Deviations (not gaps)

**LAYOUT-01 reading:** The pre-pivot "Layout" button/menu (grid 2×2 / columns preset) was removed during UAT as dead UI. LAYOUT-01 is satisfied via the per-pane split button (`_split_active_pane`) and draggable `Gtk.Paned` dividers with proportional positioning — free-layout within a worktree workspace as the requirement specifies. The removed preset menu was a pre-pivot global-layout artifact that had no meaning under the new model.

**False C-Space z zoom hint removed:** The hintbar no longer advertises a `z` chord because zoom is button-only post-pivot. The hint now correctly matches real dispatched keybindings only.
