<plan_decisions>
- **Sandbox $HOME staging (D-09):** the smoke creates its task dir via
  `task_dir = tempfile.mkdtemp(dir=os.path.expanduser("~"), prefix=".arduis-smoke-")` and removes it in
  a finally/fixture teardown. Even though no docker runs here, this models the production+UAT path and
  documents the snap constraint in code.
- **The headless smoke is autonomous (Task 1)** — collected by pytest, runs in CI/headless. It does NOT
  start GTK (the generation pipeline is GTK-free) and does NOT invoke docker. It asserts:
  1. A real base `docker-compose.yml` staged in the sandbox task dir (a minimal valid compose with a
     `web` service publishing `8080:80` and a `db` publishing `5432:5432`).
  2. Feed a captured `config --format json` model (reuse tests/fixtures/compose_config.json from Plan
     01, or a local one matching the staged base) -> `parse_published_ports` -> `assign_ports(offset=1000,
     probe=all-free)` -> `override_bytes` -> WRITE the bytes to `<task_dir>/docker-compose.override.yml`.
  3. Read the written override file back: assert it CONTAINS the literal `ports: !override`, contains the
     offset host port (e.g. `9080:80`), and does NOT contain the base published port for that mapping
     (no standalone `8080:80`). This is the criterion-2 "whole ports list rewritten" proof on real disk.
  4. argv: assert `up_argv`/`down_argv`/`config_argv`/`ls_argv` produce the exact list-form argv with the
     task_dir base+override -f paths, `-p <sanitized project>`, down's `--remove-orphans --volumes`, ls's
     `name=arduis` filter.
  5. Probe: `assign_ports` with a mock probe that returns False at offset step 1000 and True at 2000 ->
     the host ports are base+2000 (the offset-bump path); a mock that always returns False raises
     PortAssignmentError.
  6. State round-trip: `write_container_state(task_dir, ContainerState(project, True, port_map))` then
     `load_container_state(task_dir)` returns an equal state (badges/URLs stable across restart).
- **No GTK in the smoke.** The window wiring (toggle/badges/teardown) is exercised by the live UAT
  (real docker) — a headless broadway GTK smoke cannot meaningfully bring up containers, and the research
  is explicit that real-container behavior is a LIVE UAT item, not CI-portable. (If a future degrau wants
  a broadway GTK smoke of the toggle UI sans-docker, that is deferred — not in this plan.)
- **The live UAT (Task 2, checkpoint:human-verify)** is the human gate. It requires a real project with a
  root `docker-compose.yml` under `$HOME` and a running docker. The checklist walks the 5 criteria.
- **VALIDATION.md is the contract:** this plan's smoke implements the Wave-0 gaps + the requirement→test
  map from 07-VALIDATION.md. Cross-check that every "unit"/"smoke" row there has a corresponding assertion
  here or in Plan 01/02's tests; every "host-only live" row is in the UAT checklist.
</plan_decisions>
