<objective>
Build the GTK-free `containerstate.py` — the per-task container state persistence layer. It reads
and atomically writes `<task_dir>/arduis.container.toml` (the COMPOSE_PROJECT_NAME, the on/off flag,
and the resolved base->host port map) so badges and re-enable stay deterministic across app
restarts, and reads the `[containers].port_offset` config from the user-level `arduis.toml`. Disk is
the source of truth (matching 03.2's no-app-state-file model), rediscovered by the startup scan. No
`gi` import: the suite runs headless on the host.

Purpose: This is the durable record criterion 3 (stable ports/badges across restart) and criterion 4
(reconcile after crash) depend on. Keeping the read tolerant (a corrupt file no-ops to "not
isolated", never crashes task creation) and the write atomic (tmp+os.replace, can't corrupt a
half-written record) is what makes the feature survive crashes and restarts without surprises. It is
fully disjoint from Plan 01 (different files) so both run in Wave 1.
Output: `src/arduis/containerstate.py`, `tests/test_containerstate.py`; full suite green
(baseline 274 + new).

def write_theme(path: str, name: str) -> None:
    try:
        with open(path, "rb") as fh:
            data = tomllib.load(fh)
        if not isinstance(data, dict): data = {}
    except (OSError, tomllib.TOMLDecodeError):
        data = {}
    data.setdefault("theme", {})["name"] = name
    text = _serialize(data)
    try:
        d = os.path.dirname(path) or "."
        os.makedirs(d, exist_ok=True)
        fd, tmp = tempfile.mkstemp(dir=d, prefix=".arduis-cfg-")
        try:
            with os.fdopen(fd, "w", encoding="utf-8") as fh:
                fh.write(text)
            os.replace(tmp, path)
        except OSError:
            try: os.unlink(tmp)
            except OSError: pass
    except OSError:
        pass  # best-effort persistence
```

def load_theme_name(path: str) -> str:
    try:
        with open(path, "rb") as fh:
            data = tomllib.load(fh)
    except (OSError, tomllib.TOMLDecodeError):
        return _DEFAULT_THEME
    section = data.get("theme")
    if not isinstance(section, dict):
        return _DEFAULT_THEME
    name = section.get("name")
    if not isinstance(name, str) or not name.strip():
        return _DEFAULT_THEME
    return name.strip()
```

<plan_decisions>
- **`ContainerState` dataclass:** `@dataclass class ContainerState: project_name: str = ""; enabled: bool = False; ports: dict[str, list[dict]] = field(default_factory=dict)`.
  `project_name == "" and enabled is False and ports == {}` is the no-op default (a task with no
  isolation), returned for any missing/garbage/wrong-type file.
- **File path:** the caller passes `task_dir`; the file is `<task_dir>/arduis.container.toml`. Provide
  `state_path(task_dir) -> str` = `os.path.join(task_dir, "arduis.container.toml")` so window + tests
  agree on the name.
- **`load_container_state(task_dir)` tolerance (D-07, CONT-04):** open `state_path(task_dir)`; on
  `OSError`/`TOMLDecodeError` -> `ContainerState()`. `data.get("container")` not a dict ->
  `ContainerState()`. Read `project_name` (str, else "") , `enabled` (bool, else False), `ports` (a
  table mapping service -> list of port dicts, else {}). For each port-list entry keep only dicts with
  int `base`/`host`/`target` and a str-or-None `host_ip`; a malformed entry is dropped (defensive). A
  partially-valid file yields the valid parts (never raises). The DOMINANT case (no file) -> the no-op default.
- **Port-map serialization shape (must match compose.assign_ports output):** the value is
  `{service: [{"base":int,"host":int,"target":int,"host_ip":str|None}]}`. Recommended TOML layout: a
  `[container]` table with `project_name`/`enabled`, then ONE array-of-tables per service under
  `[[container.ports.<service>]]` rows, OR — cleaner for round-trip — store the whole map as a TOML
  inline structure the executor designs. The EXACT layout is the executor's choice PROVIDED a
  `tomllib.load` of the written file round-trips back to an equal ContainerState (pinned by a test).
  Recommend: `[container]\nproject_name = "..."\nenabled = false\n` then per service
  `[[container.ports]]\nservice="web"\nbase=8080\nhost=9080\ntarget=80\nhost_ip="127.0.0.1"\n`
  (a FLAT array-of-tables list with an explicit `service` key — simplest to emit and parse; omit
  `host_ip` when None). load_container_state rebuilds the `{service: [...]}` dict from that flat list.
- **`write_container_state(task_dir, state)` (D-07, CONT-04):** serialize ContainerState to the TOML
  text (the layout above), then write atomically EXACTLY like `appconfig.write_theme`:
  same-dir `tempfile.mkstemp`, `os.fdopen` write, `os.replace`, `makedirs` the parent
  (`task_dir` already exists in production but makedirs is harmless), swallow OSError best-effort.
  A boolean serializes as TOML `true`/`false`; strings are quoted with the appconfig `_fmt_scalar`
  escaping (`\\` and `"` escaped); int values bare.
- **`read_port_offset(config_path)` (D-06, CONT-03):** tolerant read of `~/.config/arduis/arduis.toml`
  `[containers].port_offset`; return the int if present and an int, else default `1000`. Missing file/
  garbage/`[containers]` not a table/`port_offset` not an int -> 1000. (Note the section is
  `[containers]` — plural — in the USER config, distinct from this task file's `[container]` table.)
- **GTK-free (the discipline):** stdlib only (`os`, `tempfile`, `tomllib`, `dataclasses`). NEVER `gi`.
  The test asserts `"import gi" not in source` and `"from gi" not in source`.
- **Sandbox HOME for tests (D-09):** tests use `tmp_path` for the task file and a `tmp_path` config
  file — NOT a real `~/.config`. (No docker is invoked here, so the snap `$HOME` constraint does not
  apply to THIS plan's tests — only to the live-docker smoke in Plan 05. Still, never touch the real
  user config.)
</plan_decisions>
